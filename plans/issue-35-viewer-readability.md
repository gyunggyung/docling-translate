# Issue #35: 뷰어 가독성 향상

## 목표

HTML 뷰어의 가독성을 높이기 위해 문장 단위 분절을 지양하고 원문의 문단 구조를 유지하며, 원문/번역문 대조 기능을 개선합니다. 또한 `docling`의 `TextItem` 레이블을 활용하여 제목, 리스트, 본문을 구분하고 페이지 번호 표시 방식을 최적화합니다.
**특히, 사용자가 번역의 정확성을 검증할 수 있도록 '문장 단위 대조(Inspection Mode)' 기능을 제공합니다.**

## 배경

현재 HTML 뷰어는 `TextItem` 내의 문장들을 모두 분리하여 별도의 행(`div.row`)으로 출력하고 있습니다. 이로 인해 문맥이 끊겨 보이고 가독성이 떨어집니다. 하지만 사용자는 번역 누락이나 오역을 확인하기 위해 **문장 단위의 1:1 비교** 기능도 여전히 필요로 합니다.

## 제안 변경 사항

### 1. `main.py` 로직 개선

#### [MODIFY] [main.py](file:///c:/github/docling-translate/main.py)

**변경 내용**:
1.  **`DocItemLabel` 활용**:
    - `docling_core.types.doc.DocItemLabel`을 사용하여 텍스트 아이템의 유형을 식별합니다.
    - `TITLE`, `SECTION_HEADER`: 문장 분리 없이 헤더 스타일(`<h3>` 등)로 출력.
    - `LIST_ITEM`: 리스트 스타일로 출력.
    - `TEXT`, `PARAGRAPH`: **문단 단위**로 묶어서 출력하되, 내부에는 **문장 단위 `<span>`**을 유지.

2.  **HTML 구조 및 데이터 매핑**:
    - `div.content-block` (문단 컨테이너)
        - `div.src-block` (원문 문단) -> 내부: `<span id="src-1">...</span>`
        - `div.tgt-block` (번역문 문단) -> 내부: `<span id="tgt-1" data-src="src-1">...</span>`
    - 각 문장 `<span>`에 고유 ID 부여하여 1:1 매핑 유지.

3.  **페이지 번호 최적화**:
    - 페이지 변경 시점에만 `<div class="page-marker">Page X</div>` 삽입.

## 예상 효과

- **가독성**: 문단 구조 유지로 읽기 편함.
- **신뢰성**: 필요시 언제든 '검수 모드'로 전환하여 문장 단위 1:1 매핑을 꼼꼼히 확인할 수 있음.

## 주의사항

- 검수 모드에서 원문/번역문 길이가 크게 다를 경우 좌우 줄이 정확히 맞지 않을 수 있음 (하지만 순서는 일치하므로 확인 가능).
