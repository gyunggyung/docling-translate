# Issue #6: 폴더(일괄) 번역 기능 구현 계획

## 목표 (Goal)
사용자가 파일 하나가 아닌, 폴더 전체 또는 여러 개의 PDF 파일을 한 번에 번역할 수 있도록 기능을 확장합니다.
CLI와 웹 뷰어(`app.py`) 모두에 일괄 처리 기능을 적용합니다.

## 변경 사항 (Proposed Changes)

### 1. `main.py` (CLI & Core Logic)
- **입력 처리 변경**: `pdf_path` 인자가 디렉토리일 경우, 해당 디렉토리 내의 모든 `.pdf` 파일을 찾아 리스트업합니다.
- **반복 처리**: 추출된 PDF 파일 리스트를 순회하며 `process_document`를 호출합니다.
- **에러 처리**: 하나의 파일이 실패하더라도 멈추지 않고 다음 파일로 넘어가도록 예외 처리를 강화합니다.

### 2. `app.py` (Web Viewer)
- **다중 파일 업로드**: `st.file_uploader`의 `accept_multiple_files=True` 옵션을 활성화합니다.
- **일괄 처리 로직**: 업로드된 모든 파일에 대해 순차적으로 번역을 수행합니다.
- **결과 표시**:
    - 번역이 완료될 때마다 진행 상황(Progress Bar)을 표시합니다.
    - 히스토리에는 각 파일별로 결과가 저장되므로, 기존 히스토리 뷰어는 그대로 작동합니다.
    - (선택 사항) 이번 세션에서 번역한 파일들의 목록을 요약해서 보여줍니다.

### 3. 문서 (Documentation)
- **`README.md` & `docs/README.en.md`**:
    - CLI에서 폴더를 지정하여 실행하는 명령어 예시 추가.
    - 웹 뷰어에서 여러 파일을 선택하여 업로드하는 방법 설명 추가.

## 검증 계획 (Verification Plan)

### 1. CLI 테스트
- `samples/` 폴더(또는 테스트용 폴더)를 입력으로 주어 내부의 모든 PDF가 번역되는지 확인.
- `python main.py "samples/"` 실행.

### 2. 웹 뷰어 테스트
- 브라우저에서 "Browse files"를 클릭하고 2개 이상의 PDF를 동시에 선택하여 업로드.
- "새로 번역 시작" 클릭 시 순차적으로 번역되고, 결과 폴더가 각각 생성되는지 확인.
- 히스토리 탭에서 각각의 결과가 조회되는지 확인.
